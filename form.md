# 폼 & 입력 UX Writing 가이드

## 정의
폼은 사용자가 정보를 직접 입력하는 영역입니다.
레이블, 플레이스홀더, 힌트, 유효성 메시지가 포함됩니다.

## 구성 요소별 원칙

### 레이블 (Label)
- 명사형으로 짧게
- 필수 항목은 * 표시 + "필수" 범례 제공
- 레이블 없는 입력 필드 금지

### 플레이스홀더 (Placeholder)
- 입력 형식 예시 또는 힌트
- 레이블을 대체하지 않음 (사라지면 기억 못함)
- "입력하세요" 단독 사용 금지
- 예시 형태로 ("예: hong@email.com")

### 힌트 텍스트 (Helper Text)
- 입력 전에 미리 안내
- 형식, 조건, 이유 포함
- 1줄 이내

### 유효성 메시지 (Validation)
- 실시간 또는 포커스 아웃 시 표시
- 성공: 간단한 확인
- 오류: 무엇이 잘못됐는지 + 어떻게 고치는지

## Before / After 예시

### 레이블
| Before ❌ | After ✓ |
|-----------|---------|
| "Name" | "이름" |
| "PW" | "비밀번호" |
| "Phone Number" | "휴대폰 번호" |

### 플레이스홀더
| Before ❌ | After ✓ |
|-----------|---------|
| "이메일을 입력하세요" | "예: hong@email.com" |
| "비밀번호 입력" | "영문, 숫자 포함 8자 이상" |
| "이름" | "홍길동" |
| "전화번호를 입력해 주세요" | "010-0000-0000" |

### 힌트 텍스트
| 상황 | Before ❌ | After ✓ |
|------|-----------|---------|
| 비밀번호 | "비밀번호를 입력하세요" | "영문, 숫자, 특수문자 포함 8자 이상" |
| 닉네임 | "닉네임" | "2~10자, 특수문자 제외" |
| 생년월일 | "생년월일 입력" | "예: 19900101" |

### 유효성 메시지
| 상황 | Before ❌ | After ✓ |
|------|-----------|---------|
| 이메일 오류 | "이메일 오류" | "이메일 형식을 확인해주세요 (예: hong@email.com)" |
| 비밀번호 불일치 | "비밀번호 불일치" | "비밀번호가 서로 달라요" |
| 사용 가능 닉네임 | "사용 가능" | "사용할 수 있는 닉네임이에요 ✓" |
| 중복 닉네임 | "사용 불가" | "이미 사용 중인 닉네임이에요" |
| 필수 항목 미입력 | "필수 항목" | "이름을 입력해주세요" |

## 금지 표현
- 플레이스홀더에 "입력하세요" 단독 사용
- 레이블을 플레이스홀더로 대체
- 유효성 메시지에서 사용자 탓 표현
- 영문 레이블 단독 사용 (한국어 서비스 기준)
- "*표시는 필수항목입니다" 범례 없이 * 단독 사용
