# 토스트 메시지 UX Writing 가이드

## 정의
토스트 메시지는 사용자 액션의 결과를 짧게 알리는 비방해형 알림입니다.
화면 하단 또는 상단에 잠깐 나타났다 사라집니다.

## 핵심 원칙
- 1~2줄 이내로 작성 (최대 40자)
- 과거형 또는 완료형으로 작성
- 사용자를 주어로 쓰지 않음 (시스템이 결과를 알리는 것)
- 느낌표 사용 금지 (과도한 감정 표현)
- 완료 상황에서는 긍정적이고 간결하게

## 유형별 작성 규칙

### 성공 토스트
- 완료된 행동을 과거형으로 표현
- "~했어요" 또는 "~됐어요" 형태
- 아이콘: ✓ (체크)

### 오류 토스트
- 무슨 문제인지 한 줄로
- 해결 방법이 있으면 포함
- 아이콘: ! (경고)

### 정보 토스트
- 중립적 사실 전달
- 판단이나 감정 없이 서술
- 아이콘: i (정보)

## Before / After 예시

| 상황 | Before ❌ | After ✓ |
|------|-----------|---------|
| 저장 완료 | "저장이 완료되었습니다" | "저장됐어요" |
| 복사 완료 | "클립보드에 복사되었습니다" | "복사됐어요" |
| 삭제 완료 | "삭제가 완료되었습니다" | "삭제했어요" |
| 전송 완료 | "메시지 전송이 완료되었습니다" | "메시지를 보냈어요" |
| 오류 발생 | "오류가 발생하였습니다" | "문제가 생겼어요. 다시 시도해주세요" |
| 네트워크 오류 | "네트워크 연결 상태를 확인하세요" | "인터넷 연결을 확인해주세요" |

## 금지 표현
- "완료되었습니다" → "됐어요"로 대체
- "발생하였습니다" → "생겼어요"로 대체
- "확인하시기 바랍니다" → "확인해주세요"로 대체
- 느낌표(!) 사용 금지
- "성공적으로" 불필요한 부사 사용 금지
