# 팝업 UX Writing 가이드

## 정의
팝업은 사용자가 요청하지 않았지만 시스템이 자발적으로 표시하는
알림 또는 안내입니다. 모달보다 덜 강제적이며,
주로 프로모션·안내·권한 요청에 사용됩니다.

## 모달과의 차이
- 모달: 사용자 액션에 반응 / 결정 필요
- 팝업: 시스템 주도 / 닫기 가능 / 덜 긴급

## 유형별 작성 규칙

### 권한 요청 팝업
- 왜 이 권한이 필요한지 사용자 혜택 중심으로 설명
- "허용하면 ~할 수 있어요" 형태
- 거부 옵션도 명확하게

### 프로모션 팝업
- 혜택을 먼저, 조건은 나중에
- 과장 표현 금지
- CTA 버튼은 혜택 중심으로

### 업데이트 안내 팝업
- 무엇이 바뀌었는지 구체적으로
- "더 나아졌어요" 같은 모호한 표현 지양

## Before / After 예시

| 상황 | Before ❌ | After ✓ |
|------|-----------|---------|
| 알림 권한 | "알림을 허용하시겠습니까?" | "새 소식을 빠르게 받아볼 수 있어요. 알림을 허용할까요?" |
| 위치 권한 | "위치 정보 접근을 허용해주세요" | "내 주변 매장을 찾으려면 위치가 필요해요" |
| 프로모션 | "특별 할인 이벤트!" | "지금 가입하면 첫 달 50% 할인이에요" |
| 업데이트 | "앱이 업데이트 되었습니다" | "검색이 더 빨라졌어요. 새 버전을 확인해보세요" |
| 재방문 유도 | "오랜만이에요! 돌아오셨군요!" | "12일 만이에요. 이어서 볼까요?" |

## 금지 표현
- "~하시겠습니까?" → "~할까요?"
- 느낌표 남발 금지
- "특별", "놀라운", "대박" 등 과장 수식어 금지
- 혜택 없이 권한만 요구하는 문구 금지
