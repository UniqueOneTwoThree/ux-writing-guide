# CTA 버튼 UX Writing 가이드

## 정의
CTA(Call to Action)는 사용자에게 특정 행동을 유도하는
버튼 또는 링크 텍스트입니다.

## 핵심 원칙
- 동사로 시작
- 10자 이내 (모바일 기준)
- 버튼만 봐도 어떤 행동인지 명확하게
- 1개 화면에 주요 CTA는 1개만
- 혜택이 있으면 혜택을 버튼에 포함

## 유형별 작성 규칙

### 시작/가입 CTA
- 부담 없이 시작할 수 있다는 느낌
- "무료로", "지금" 등 수식어 활용 가능

### 구매/결제 CTA
- 금액 또는 혜택 포함 가능
- 다음 단계가 명확하게

### 확인/완료 CTA
- 행동의 결과가 보이게
- "확인" 단독 사용 지양

### 탐색/더보기 CTA
- 무엇을 더 볼 수 있는지 명확하게
- "더보기" 단독보다 구체적으로

## Before / After 예시

| 상황 | Before ❌ | After ✓ |
|------|-----------|---------|
| 회원가입 | "가입" | "무료로 시작하기" |
| 구매 | "구매" | "지금 구매하기" |
| 결제 | "확인" | "결제하기" |
| 다음 단계 | "다음" | "약관 동의하기" |
| 더보기 | "더보기" | "리뷰 더 보기" |
| 로그인 | "로그인" | "계속하기" |
| 저장 | "저장" | "변경사항 저장" |
| 탈퇴 | "탈퇴" | "서비스 탈퇴하기" |
| 공유 | "공유" | "친구에게 공유" |

## 주요/보조 버튼 쌍 예시

| 상황 | 주요 버튼 | 보조 버튼 |
|------|-----------|-----------|
| 삭제 확인 | "삭제할게요" | "취소" |
| 업데이트 | "지금 업데이트" | "나중에" |
| 권한 요청 | "허용하기" | "괜찮아요" |
| 구매 확인 | "결제하기" | "장바구니로" |

## 금지 표현
- "클릭하세요" → "탭" 또는 동사형으로
- "확인" 단독 사용 금지
- "Submit", "OK" 등 영문 단독 사용 금지
- "~하시기 바랍니다" 형태 금지
