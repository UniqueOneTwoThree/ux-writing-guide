# 모달 UX Writing 가이드

## 정의
모달은 사용자의 흐름을 잠깐 멈추고 중요한 정보를 확인하거나
결정을 요구하는 레이어입니다.

## 구성 요소
1. **제목 (Title)** — 모달의 목적을 한 줄로
2. **본문 (Body)** — 상황 설명 또는 확인 내용
3. **액션 버튼** — 주요 액션 / 보조 액션

## 제목 작성 규칙
- 10자 이내 권장
- 명사형 또는 동사형 모두 가능
- 의문형 제목은 지양 ("삭제할까요?" → "삭제하기")
- 시스템 관점이 아닌 사용자 행동 중심으로

## 본문 작성 규칙
- 3줄 이내
- 사용자가 왜 이 결정을 해야 하는지 명확하게
- 전문 용어나 시스템 언어 사용 금지
- 되돌릴 수 없는 액션은 반드시 경고 포함

## 버튼 작성 규칙
- 주요 버튼: 동사로 시작, 행동이 명확하게
- 보조 버튼: "취소" 또는 "닫기"
- "확인" 단독 사용 금지 (무슨 확인인지 불분명)
- 버튼만 봐도 어떤 행동인지 알 수 있어야 함

## Before / After 예시

| 상황 | Before ❌ | After ✓ |
|------|-----------|---------|
| 삭제 확인 제목 | "삭제 확인" | "삭제하기" |
| 삭제 확인 본문 | "정말로 삭제하시겠습니까? 삭제된 데이터는 복구할 수 없습니다." | "삭제하면 되돌릴 수 없어요. 그래도 삭제할까요?" |
| 삭제 버튼 | "확인" | "삭제할게요" |
| 로그아웃 제목 | "로그아웃" | "로그아웃하기" |
| 로그아웃 본문 | "로그아웃 하시겠습니까?" | "로그아웃하면 다시 로그인해야 해요" |
| 저장 안내 | "변경사항이 있습니다" | "저장하지 않은 내용이 있어요" |

## 금지 표현
- "~하시겠습니까?" → "~할까요?" 또는 "~하기"
- "확인" 단독 버튼 사용 금지
- "취소하시겠습니까?" → 모달 본문에 넣지 않음
- 과도한 경고 문구 (사용자를 겁주는 표현) 지양
