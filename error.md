# 오류 메시지 UX Writing 가이드

## 정의
오류 메시지는 사용자가 원하는 작업을 완료할 수 없을 때
시스템이 전달하는 안내입니다.

## 핵심 원칙
- 사용자 탓으로 돌리지 않음
- 원인 + 해결 방법을 함께 제공
- 시스템 언어(에러 코드, 기술 용어) 노출 금지
- 가능하면 다음 행동을 제시
- 짧고 명확하게 (2줄 이내)

## 오류 유형별 작성 규칙

### 입력값 오류 (Validation)
- 무엇이 잘못됐는지 구체적으로
- 올바른 형식 예시 제공
- 부드럽고 비판적이지 않게

### 시스템/네트워크 오류
- 사용자 책임이 아님을 암시
- 재시도 유도
- 지속되면 어떻게 할지 안내

### 권한/접근 오류
- 왜 접근할 수 없는지 간단히
- 해결 경로 제시

### 빈 검색결과
- 오류가 아닌 안내로 접근
- 다른 시도 방법 제안

## Before / After 예시

| 상황 | Before ❌ | After ✓ |
|------|-----------|---------|
| 비밀번호 오류 | "비밀번호가 틀렸습니다" | "비밀번호가 맞지 않아요. 다시 확인해주세요" |
| 이메일 형식 | "올바르지 않은 이메일 형식입니다" | "이메일 주소를 확인해주세요 (예: hello@email.com)" |
| 네트워크 오류 | "네트워크 오류가 발생했습니다. Error Code: 503" | "인터넷 연결을 확인하고 다시 시도해주세요" |
| 서버 오류 | "서버 오류가 발생하였습니다" | "잠깐 문제가 생겼어요. 조금 후에 다시 시도해주세요" |
| 권한 없음 | "접근 권한이 없습니다" | "이 페이지는 로그인 후 볼 수 있어요" |
| 파일 용량 초과 | "파일 사이즈 초과" | "파일이 너무 커요. 10MB 이하로 올려주세요" |
| 중복 이메일 | "이미 존재하는 이메일입니다" | "이미 사용 중인 이메일이에요. 로그인해볼까요?" |

## 금지 표현
- "오류가 발생하였습니다" → 구체적 원인으로 대체
- 에러 코드 단독 노출 (Error 404, 503 등)
- "잘못된 입력입니다" → 무엇이 잘못됐는지 명시
- "다시 시도하십시오" → "다시 시도해주세요"
- 사용자를 탓하는 표현 ("잘못 입력하셨습니다")
