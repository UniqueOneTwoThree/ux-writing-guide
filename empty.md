# 빈 상태(Empty State) UX Writing 가이드

## 정의
빈 상태는 콘텐츠가 없거나 검색 결과가 없을 때
사용자에게 표시되는 화면입니다.

## 핵심 원칙
- 오류처럼 보이지 않게
- 왜 비어있는지 맥락 제공
- 다음 행동을 명확하게 유도
- 긍정적이고 가볍게

## 유형별 작성 규칙

### 콘텐츠가 아직 없음 (첫 방문)
- 기회로 프레이밍
- 첫 액션 유도 CTA 포함

### 검색 결과 없음
- 검색어를 포함해서 안내
- 다른 시도 방법 제안

### 필터 결과 없음
- 필터 조건 때문임을 명시
- 필터 초기화 CTA 제공

### 완료/처리됨 (긍정적 빈 상태)
- 칭찬 또는 긍정 표현
- 다음 할 일 제안

## Before / After 예시

| 상황 | Before ❌ | After ✓ |
|------|-----------|---------|
| 알림 없음 | "알림이 없습니다" | "새로운 알림이 없어요" |
| 장바구니 비어있음 | "장바구니가 비어있습니다" | "아직 담은 상품이 없어요. 마음에 드는 걸 찾아볼까요?" |
| 검색 결과 없음 | "검색 결과가 없습니다" | "'OO'에 대한 결과가 없어요. 다른 키워드로 찾아볼까요?" |
| 할 일 모두 완료 | "할 일이 없습니다" | "모두 완료했어요! 오늘도 수고했어요 👏" |
| 팔로잉 없음 | "팔로잉한 유저가 없습니다" | "아직 팔로우한 사람이 없어요. 관심 있는 사람을 찾아보세요" |
| 필터 결과 없음 | "해당 조건의 결과가 없습니다" | "이 조건에 맞는 결과가 없어요. 필터를 바꿔볼까요?" |

## 금지 표현
- "데이터가 없습니다" → 맥락 있는 표현으로
- "결과가 존재하지 않습니다" → 자연스러운 표현으로
- 단순 "없음" 상태만 전달하고 다음 행동 없음
- 오류처럼 느껴지는 표현
